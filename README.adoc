= audiowmark - Marcação de Áudio

== Descrição

audiowmark é uma solução Open Source (GPL) para marcação de áudio.

Um arquivo de áudio é lido pelo software, e uma mensagem de 128 bits é armazenada como watermark (marca d’água) no arquivo de saída. Para ouvintes humanos, os arquivos soam praticamente iguais.

No entanto, a mensagem de 128 bits pode ser recuperada do arquivo de áudio de saída. Nossos testes mostram que, mesmo se o arquivo for convertido para mp3 ou ogg (com taxa de 128 kbit/s ou superior), a marca d’água geralmente pode ser recuperada sem problemas. O processo de recuperação da mensagem não precisa do arquivo original (decodificação cega).

Internamente, audiowmark usa o algoritmo patchwork para esconder os dados no espectro do áudio. O sinal é dividido em frames de 1024 amostras. Para cada frame, algumas amplitudes das bandas de frequência de uma FFT de 1024 valores são aumentadas ou diminuídas ligeiramente de forma pseudo-aleatória, o que pode ser detectado posteriormente. O algoritmo é inspirado em:

Martin Steinebach: Digitale Wasserzeichen für Audiodaten.
Darmstadt University of Technology 2004, ISBN 3-8322-2507-2

Tutorial em português para instalação no Windows

== Adicionando uma Watermark

Para adicionar uma marca d’água ao arquivo in.wav com uma mensagem de 128 bits (especificada como hex string):

[subs=+quotes]
....
$ audiowmark add in.wav out.wav 0123456789abcdef0011223344556677
Input: in.wav
Output: out.wav
Message: 0123456789abcdef0011223344556677
Strength: 10

Time: 3:59
Sample Rate: 48000
Channels: 2
Data Blocks: 4
....

Se você quiser usar audiowmark em qualquer aplicação séria, leia a seção <<rec-payload>> sobre como gerar a mensagem de 128 bits. Normalmente, esses bits devem ser um hash ou HMAC de algum tipo.

As opções mais importantes para adicionar uma marca d’água são:

--key <filename>::
Usa uma chave de watermark a partir do arquivo <filename> (veja <<key>>).

--strength <s>::
Define a força da marca d’água (veja <<strength>>).

== Recuperando uma Watermark

Para obter a mensagem de 128 bits do arquivo com marca d’água, use:

[subs=+quotes]
....
$ audiowmark get out.wav
pattern 0:05 0123456789abcdef0011223344556677 1.358 0.059 A
pattern 0:57 0123456789abcdef0011223344556677 1.426 0.050 B
pattern 0:57 0123456789abcdef0011223344556677 1.392 0.055 AB
pattern 1:49 0123456789abcdef0011223344556677 1.453 0.074 A
pattern 2:40 0123456789abcdef0011223344556677 1.359 0.080 B
pattern 2:40 0123456789abcdef0011223344556677 1.406 0.077 AB
pattern all 0123456789abcdef0011223344556677 1.399 0.035
....

O output de audiowmark get é projetado para ser lido por máquinas. Cada linha que começa com pattern contém uma mensagem decodificada.

O primeiro campo é o timestamp indicando a posição do bloco de dados.

O segundo campo é a mensagem decodificada.

O terceiro campo é a pontuação de sincronização (quanto maior, melhor).

O quarto campo é o erro de decodificação (quanto menor, melhor).

O quinto campo indica o tipo de bloco: A, B ou AB.

== Chave de Watermark

Uma chave de watermark garante que os bits da mensagem não possam ser recuperados por terceiros. A chave controla todos os parâmetros pseudo-aleatórios do algoritmo.

Exemplo de criação de chave:

audiowmark gen-key test.key

Usando a chave nos comandos add/get:

audiowmark add --key test.key in.wav out.wav 0123456789abcdef0011223344556677
audiowmark get --key test.key out.wav

== Força da Watermark

O parâmetro strength define o quanto a marca altera o sinal de entrada.

Maior força: mais audível, mais robusta.

Menor força: menos audível, menos robusta.

Exemplo:

audiowmark add --strength 15 in.wav out.wav 0123456789abcdef0011223344556677

== Recomendações para o Payload da Watermark

O payload de 128 bits deve ser um hash ou HMAC.

Exemplo de uso:

STRING='Artist:Alice|Title:Dreams|User:John Smith'
MSG=echo -n "$STRING" | sha256sum | head -c 32
audiowmark add --key my.key song.wav song.wm.wav $MSG

== Detecção de Velocidade (Speed Detection)

Se o áudio marcado for reproduzido mais rápido ou mais lento, a detecção pode falhar. Para corrigir:

audiowmark get out1.wav --detect-speed

Existem duas versões: --detect-speed e --detect-speed-patient (mais precisa, mas mais lenta).

== Arquivos de Vídeo

Use videowmark para adicionar marca d’água à trilha de áudio de vídeos:

videowmark add in.avi out.avi 0123456789abcdef0011223344556677
videowmark get out.avi

== Entrada e Saída via Stream

É possível processar streams de áudio:

Saída como stream:

audiowmark add in.wav - 0123456789abcdef0011223344556677 | play -

Entrada como stream:

cat in.wav | audiowmark add - out.wav 0123456789abcdef0011223344556677

Pipe WAV longo:

cat in.mp3 | madplay -o wave:- - | audiowmark add --input-format wav-pipe --output-format rf64 - out.wav f0

== Streams Raw

Para streams sem header WAV, é necessário informar manualmente:

--input-format raw::
--output-format raw::
--raw-rate <rate>::
--raw-input-bits <bits>::
--raw-channels <channels>::

== Outras Opções de Linha de Comando

--output-format rf64:: (permite arquivos WAV > 4GB)
--q, --quiet:: (desativa mensagens)
--strict:: (ativa checagem rigorosa de erros)

== HTTP Live Streaming (HLS)

HLS permite marcar o áudio enquanto ele é transmitido:

Preparar segmentos:

ffmpeg -i video.mp4 -f hls -master_pl_name replay.m3u8 -c:a aac -ab 192k -var_stream_map "a:0,agroup:aud v:0,agroup:aud" -hls_playlist_type vod -hls_list_size 0 -hls_time 10 vs%v/out.m3u8

Preparar áudio para watermark:

audiowmark hls-prepare vs0 vs0prep out.m3u8 video.mp4

Adicionar watermark ao segmento:

audiowmark hls-add vs0prep/out5.ts send5.ts 0123456789abcdef0011223344556677
